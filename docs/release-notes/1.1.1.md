# 1.1.1 — re-export of message-propagation types

Patch release. No runtime change, no breaking change.

## Problem

1.1.0 introduced two public types that consumers needed to import when wiring the new message-propagation feature:

- `HmacPropagateTargetStore` — the `"http" | "message"` literal accepted by `propagateClientToApis({ targetStore })`.
- `HmacMessageAuthBridge` — the structural shape required by `initializeHmacHttpAuth({ messageAuth })`.

Both were declared in `src/core/types.ts` with `export`, but `src/index.ts` only re-exported a curated subset of the type list, and these two were left out. Result:

```ts
import type { HmacPropagateTargetStore } from "@naskot/node-hmac-auth";
//                ^^^^^^^^^^^^^^^^^^^^^^^^^^
// TS2305: Module '"@naskot/node-hmac-auth"' has no exported member 'HmacPropagateTargetStore'.
```

Consumers had to either (a) reach into the deep import path (not part of the public surface) or (b) inline the literal `"http" | "message"` and a hand-rolled bridge shape, duplicating the lib's source of truth.

## Fix

`src/index.ts` now re-exports the two missing types alongside the rest of the public type surface:

```ts
export type {
  // ...
  HmacMessageAuthBridge,
  HmacPropagateTargetStore,
  // ...
} from "./core/types.js";
```

## Migration

Drop any inline `"http" | "message"` literal or hand-rolled `HmacMessageAuthBridge` shape and pull from the package directly:

```ts
import type { HmacPropagateTargetStore, HmacMessageAuthBridge } from "@naskot/node-hmac-auth";
```

No code change required if you stay on the inline literal — 1.1.1 is fully additive.
